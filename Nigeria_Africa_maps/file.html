What is DHIN FHIR Sandbox?

Open Source FHIR Conformance Testing for Digital Health

The DHIN FHIR Testing Sandbox is an open-source platform for building, running, and sharing automated tests for FHIR conformance. Designed to support real-world digital health use cases, the DHIN Sandbox empowers health IT developers, vendors, and innovators to test their systems against core FHIR standards and interoperability requirements.

Get Started with Testing

The DHIN Sandbox includes curated test kits built to validate FHIR R4's against leading implementation guides and digital health interoperability standards. These kits simulate various care scenarios, supporting validation of clinical resources such as Patient, Observation, and Encounter.

Build and Share Custom Tests

DHIN's FHIR Sandbox is as adaptable as FHIR itself. Developers can design and deploy custom test scripts tailored to unique digital health use cases. The platform supports flexible test creation using modular tools and standardized testing workflows.

Run DHIN Test Kits Locally or Remotely

The DHIN Sandbox supports both cloud-hosted and local deployments. Each DHIN Test Kit is a standalone application that targets specific FHIR implementation guides or interoperability goals. You can download and run test kits within your local environment, or deploy them securely within enterprise systems.

Integrate into Your CI/CD Workflow

With a full-featured API, the DHIN Sandbox allows integration into existing development workflows, testing environments, or CI/CD pipelines. Automate FHIR conformance checks as part of your software lifecycle.

More Than Just Tests

The DHIN Sandbox includes additional tools and utilities—such as reference FHIR applications, test data generators, and patient simulators—to support end-to-end testing. These tools help you go beyond conformance and ensure your health IT system behaves as expected in real-world scenarios.
